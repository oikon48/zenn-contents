# Oikonの記事構成パターン

このドキュメントは、Oikonの技術記事の典型的な構成パターンをまとめたものです。

## 基本構成テンプレート

```markdown
---
title: "具体的でわかりやすいタイトル"
emoji: "📱"
type: "tech"
topics: [関連トピック]
published: true
published_at: "YYYY-MM-DD HH:MM"
---

[注意書き（Optional）]

[自己紹介・背景]

[記事を書くきっかけ・動機]

## 使用したツール・サービス（Optional）

[ツールリストと簡単な説明]

## [メイントピック]の全体像

[図や概要の説明]

### ステップ1: [具体的な内容]

[詳細な説明]

### ステップ2: [具体的な内容]

[詳細な説明]

...

## まとめ

[要点の再掲]

## Xフォローしてくれると嬉しいです

[フォローリンク]

## 参考文献

[引用したリンク]
```

## パターン別の構成

### パターン1: ワークフロー解説型

**用途**: 開発フロー、ツール連携、実践的な手順の説明

**構成**:
1. 導入（自己紹介・背景・きっかけ）
2. 使用したツール・サービスのリスト
3. ワークフロー全体像（図解）
4. 各ステップの詳細解説
5. まとめ（ワークフロー図の再掲 + 要点）
6. フォロー誘導
7. 参考文献

**例**: "iPhoneだけでiOSアプリ開発するワークフロー"

**特徴**:
- 全体像の図を最初と最後に提示
- ステップごとに`##`または`###`で区切る
- 各ステップで具体的なコマンドや画面を示す
- 比較検討の結果や個人的な感想を含める

### パターン2: アップデート・新機能解説型

**用途**: ツールのバージョンアップ、新機能の詳細解説

**構成**:
1. 導入（自己紹介・アップデート概要）
2. アップデートの概要（変更点リスト）
3. 各機能の詳細解説
4. まとめ（全体の感想）
5. イベント告知（Optional）
6. フォロー誘導
7. 参考文献

**例**: "Claude Code 2.0.0のメジャーアップデートについて"

**特徴**:
- 冒頭に変更点の箇条書きリスト
- 各機能ごとにセクション分け
- スクリーンショットやGIFで視覚的に説明
- 実際に試した結果や注意点を明記

### パターン3: ツール統合・連携型

**用途**: 複数ツールの連携方法、エコシステムの構築

**構成**:
1. 導入（自己紹介・背景）
2. 運用の流れ（全体像）
3. 前提条件・必要なツール
4. 各ステップの詳細（セットアップ、設定、活用法）
5. 運用のまとめ（フロー図の再掲）
6. 全体のまとめ（コツや知見）
7. フォロー誘導
8. 参考文献

**例**: "ObsidianとClaude Codeを使ったドキュメント活用"

**特徴**:
- 運用フローを冒頭とステップ解説後に提示
- セットアップ手順を丁寧に解説
- 設定ファイルやコマンドの具体例
- 個人的な試行錯誤のプロセスを共有

### パターン4: ツール紹介・レビュー型

**用途**: 新しいツールの紹介、複数ツールの比較

**構成**:
1. 導入（自己紹介・ツールとの出会い）
2. ツール概要
3. 主な機能
4. 実際の使用例
5. 他ツールとの比較（Optional）
6. まとめ（向いている人、おすすめポイント）
7. フォロー誘導
8. 参考文献

**特徴**:
- 実際に使った感想を中心に
- スクリーンショット豊富
- メリット・デメリットを明示
- 具体的な使用例を提示

## セクション別の書き方

### 導入部分

**パターンA: 経験ベース**
```markdown
Oikonです。普段はAIツール、特にClaude Codeで遊んでいます。

先日、[イベント名]があり[何をしたか]。[その結果や反響]があったので、
記事にしようと思いました。

[経験の浅さへの言及（Optional）]
```

**パターンB: 問題提起ベース**
```markdown
Oikonです。[職業や立場]。

[トピック]が最近注目を集めています。[背景説明]。

[自分の取り組み]を3ヶ月ほど前から試行錯誤しながら使っています。
ようやく自分の中でしっくり来る運用が固まってきたので、その方法を共有します。
```

### 注意書き（Optional）

```markdown
:::message
[日付]時点の[バージョン]の内容です
:::
```

または

```markdown
:::message
この記事は人力で書いてます。
:::
```

### 使用ツール・サービスのリスト

```markdown
## 使用したツール・サービス

今回、利用したツール・サービスは以下の通りです：

- ツール1（説明）
- ツール2（説明）
- ツール3 (Optional)

[ツールの選定理由や組み合わせの説明]
```

### 全体像の提示

```markdown
## [トピック]全体像

![図のタイトル](/images/記事名/ファイル名.png)

実際に運用している[システム/ワークフロー]です。以下の[N]つのステップで構成されています

1. ステップ1: 簡単な説明
2. ステップ2: 簡単な説明
...

それぞれのステップについて詳しく紹介します
```

### ステップ詳細の解説

```markdown
### Step N. [ステップ名]（[使用ツール]）

[ステップの概要説明]

**方法1: [具体的な方法名]**

[方法の詳細説明]

[公式ドキュメントへのリンク]

[具体的な手順]

![スクリーンショット](/images/記事名/ファイル名.png)

**方法2: [別の方法]（Optional）**

[別の方法の説明]

[個人的な比較・感想]
```

### コマンド・コード例の提示

````markdown
[前後の説明文]

```bash
# コメントで説明
コマンド
```

[実行結果の説明や注意点]
````

### 注意点やTipsの提示

```markdown
:::message
[無料プランの制限]や[重要な設定]など
:::
```

### まとめセクション

```markdown
## まとめ

![全体図の再掲（Optional）](/images/記事名/ファイル名.png)

改めて[トピック]を再掲します。
**今回は、[主題]を紹介しました**。

**[要点のまとめ]**:

1. ポイント1
2. ポイント2
...

[総括的なコメント]
```

### フォロー誘導

```markdown
## Xフォローしてくれると嬉しいです

[イベント告知（Optional）]

Xでも情報発信しているので、フォローしていただけると励みになります！

https://x.com/oikon48
```

または

```markdown
## 𝕏フォローしてくれると嬉しいです！

𝕏でも情報発信しているので、フォローしていただけると励みになります！

https://x.com/oikon48
```

### 参考文献

```markdown
## 参考文献

https://公式サイト1

https://参考記事1

https://参考記事2

https://自分の過去記事
```

## 記事執筆のチェックリスト

記事を書く際は以下をチェック：

- [ ] フロントマターに適切なemoji、topics、published_atが設定されている
- [ ] 自己紹介と記事を書くきっかけが明記されている
- [ ] 全体像を図で示している（該当する場合）
- [ ] 各ステップ・機能が見出しで明確に区切られている
- [ ] スクリーンショットやコード例が豊富
- [ ] 実際に試した結果や個人的な感想が含まれている
- [ ] 注意点やTipsが明示されている
- [ ] まとめセクションで要点が再掲されている
- [ ] フォローリンクと参考文献が記載されている
- [ ] リンクが全て有効
- [ ] 画像パスが正しい（/images/記事名/ファイル名.png形式）

## 画像の扱い

### 画像の配置

- パス: `/images/[記事のslug]/[ファイル名].png`
- 例: `/images/dev-from-mobile/iphone-workflow.png`

### 画像のサイズ指定

```markdown
![alt text](/images/folder/file.png =400x)
```

### 画像の説明

- 画像の前に説明文を入れる
- alt textは簡潔に

## リンクの扱い

### 外部リンク

```markdown
https://example.com
```

または

```markdown
[リンクテキスト](https://example.com)
```

### 注釈付きリンク

```markdown
詳しいことは公式ドキュメントに書いてあります。

https://docs.example.com
```

### ツイートの埋め込み

```markdown
https://x.com/username/status/1234567890
```

### Speaker Deckの埋め込み

```markdown
https://speakerdeck.com/username/slide-name
```
